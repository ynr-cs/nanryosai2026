# Home Page (`index.html`) 開発コンテキスト

本ドキュメントは、トップページ (`index.html`) の開発ガイドラインです。

## 1. 概要

- **役割**: アプリの玄関口（ランディングページ）。
- **目的**: ユーザーに「祭りの高揚感」を与えつつ、必要な情報へ直感的にアクセスさせる。
- **デザイン**: Vibrant（鮮やか）かつ Glassmorphism（すりガラス）を取り入れたモダンな UI。

## 2. 機能・コンテンツ詳細

### 2.1 ヒーローセクション (Hero Section)

- **ビジュアル**: 没入感のあるグラデーション背景、浮遊するオーブ装飾。
- **コンテンツ**:
  - イベントバッジ: 日付などを表示。
  - **大見出し**: `Nanryosai 2026`（グラデーション文字）。
  - スローガン: 今年のテーマ。

### 2.2 カウントダウン (Countdown)

- **機能**: 現在時刻に応じたフェーズ切り替えと演出 (`index.html` 内 JS ロジック)。
  1.  **通常**: 開催まで 24 時間以上。標準表示。
  2.  **直前 (<24h)**: 開催まで 24 時間未満。テキストが赤くなり、鼓動アニメーション (`heartbeat`) が適用される。
  3.  **開催中**: 2026/9/11 9:00 - 9/12 18:00 (後夜祭含む)。カウントダウンが非表示になり、「NOW HAPPENING / 開催中」の特大バナーが表示される。
  4.  **終了**: 9/12 18:00 以降。「THANK YOU / 終了しました」が表示される。
- **デザイン**:
  - Glassmorphism スタイルのカードに Days / Hours / Minutes / Seconds をグリッド表示（通常・直前）。
  - 開催中・終了時は専用のアニメーション付きバナーを表示。

### 2.3 リアルタイム情報 (Real-time Info)

- **現状**: **Dummy UI** として実装済（2025/01/09 時点）。
- **内容**: 体育館、音楽室、視聴覚室の 3 会場の状況を表示。
- **レイアウト**: 横一列のカード配置。
- **将来の実装予定**:
  - **Cloud Functions + Storage**: 管理画面から JSON を更新し、読み込む仕組みを採用予定。
  - **Admin**: パスワード認証付きの専用更新ページ (`admin/stage-editor.html`) を作成予定。

### 2.4 ダッシュボード (Shortcuts) - _Coming Soon_

- **企画一覧**: 参加団体の出し物一覧へ遷移。
- **ステージ**: タイムテーブルへ遷移。
- **モバイルオーダー**: 模擬店の注文ページへ遷移。

### 2.5 モバイルオーダー推奨 (Mobile Order Promo)

- **役割**: モバイルオーダー機能（`pos/mobile-order.html`）への誘導。
- **デザイン**: 鮮やかなグラデーションと Glassmorphism を用いた強調カード。
  - **重要**: コンテンツ量に応じて高さが伸縮するよう `min-height` で制御。
- **CTA**: "Start Ordering" ボタンでオーダー画面へ遷移。
- **追加リンク**: 「システムの安全性について」へのリンク (`.promo-link-sub`) を配置。

### 2.6 企画・発表ハイライト (Highlights)

- **役割**: ユーザーに「面白そう」と思わせる興味喚起エリア。
- **デザイン**: 垂直自動スクロールリスト (Auto-scrolling Vertical Loop)。
  - **横幅制限**: 「開催概要」セクションと同等の **max-width: 800px** に制限し、全体を中央揃え (`margin: 0 auto`) に配置。これにより、各セクション間での視覚的な一貫性を確保。
  - **高さ固定 (380px)** のコンテナ内で、カードがゆっくりと自動で上に流れ、無限ループする。
  - ユーザーがホバーまたはタッチすると自動スクロールは一時停止し、手動でスクロール可能。
  - Scroll Snap は無効化し、スムーズな移動を実現。
- **コンテンツ**: 各クラスや部活の企画カード（サムネイル画像、団体名、企画名、場所、ジャンルタグ）。
- **実装**: `main/data/data.js` から `projectData` を読み込み、JS で動的に生成。
- **インタラクション**: 各カードはタップ可能で、詳細への遷移などを想定。上下スクロールで閲覧。

### 2.7 データ管理 (Data Management)

- **ファイル**: `main/data/data.js`
- **内容**: 企画データ (`projectData`) やステージデータ (`stageData`) を一元管理。
- **目的**: HTML とデータを分離し、更新性を向上。

### 2.8 開催概要 (Overview)

- **役割**: 開催日時、場所、注意事項などの基本情報を提供。
- **コンテンツ**:
  - 詳細テーブル（日時、場所、注意事項リスト）。
  - ※以前あった画像カードおよびテーマテキストは削除済み。
- **デザイン**:
  - `style.css` に定義された Glassmorphism デザイン (`.overview-card`, `.overview-table`) を適用。
  - ユーザーが情報をコピーできるよう、例外的に `user-select: text` を有効化。

## 3. 実装要件

- **App Shell 統合**: `app-shell.js` によりヘッダー・フッターを動的に注入。
- **レスポンシブ**: モバイルファーストだが、PC 閲覧時も間延びしないよう `max-width` などを適切に設定。
- **CSS**: `style.css` にて一元管理。変数を活用し、ダークモードにも完全対応する。
