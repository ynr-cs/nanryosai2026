# Home Page (`index.html`) 開発コンテキスト

本ドキュメントは、トップページ (`index.html`) の開発ガイドラインです。

## 1. 概要

- **役割**: アプリの玄関口（ランディングページ）。
- **目的**: ユーザーを主要コンテンツ（企画、ステージ、モバイルオーダー）へ素早く誘導する。
- **デザイン**: 祭りの高揚感を感じさせる「没入感」のあるヒーローセクションと、実用的なナビゲーションカードを組み合わせる。

## 2. 機能・コンテンツ詳細

### 2.1 ヒーローセクション (Hero Section)

- **ビジュアル**: グラデーション背景と大きなイベントタイトル (`Nanryosai '26`)。
- **コピー**: 今年のテーマやキャッチコピーを配置。
- **動的要素**: (将来的に) 残り時間のカウントダウンや、現在開催中のイベントを表示する可能性あり。

### 2.2 ダッシュボード (Shortcuts)

- **カードレイアウト**: グリッドレイアウトで主要機能へのリンクを表示。
  1.  **企画一覧**: 参加団体の出し物一覧へ遷移。
  2.  **ステージ**: 体育館・中庭ステージのタイムテーブルへ遷移。
  3.  **モバイルオーダー**: 模擬店の注文ページへ遷移（強調表示）。

### 2.3 ニュース / お知らせ (News Feed) - _Future_

- **用途**: 運営からの緊急連絡（雨天時の変更など）やトピックを表示。
- **データソース**: Firestore `news` コレクション（想定）。

## 3. 実装要件

- **App Shell 統合**: ヘッダー（ハンバーガーメニュー）、ボトムナビゲーションを含む。
- **レスポンシブ**: PC 表示時は最大幅を制限 (`max-width: 600px` 等) し、中央寄せでスマホアプリ風の比率を維持する（App Shell 共通仕様）。
- **PWA 対応**: Web App Manifest と連携し、ホーム画面に追加した際のアプリアイコンや起動画面を設定。
