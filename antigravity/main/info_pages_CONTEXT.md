# Info Pages (`map.html`, `access.html`, `about.html`, `about-us.html`) 開発コンテキスト

本ドキュメントは、静的な情報提供ページの開発ガイドラインです。

## 1. 校内マップ (`map.html`)

- **役割**: 来場者が迷わないための地図機能。
- **実装**:
  - **画像ビューア**: フロアごとの校内図（画像）を表示。
  - **ピンチイン・アウト**: `Panzoom` ライブラリ等を使用し、スマホでの拡大縮小に対応。
  - **ピン機能 (Future)**: 企画一覧からマップへ遷移した際、該当場所をピンで強調表示。

## 2. アクセス (`access.html`)

- **役割**: 学校までの交通案内。
- **内容**:
  - 最寄り駅からの徒歩ルート。
  - バスの時刻表リンク。
  - Google Maps 埋め込み。
- **注意点**: 来場者向け (`guest` モード) のみ表示するなど、対象に応じた出し分け（生徒には不要かもしれないが、保護者には必要）。

## 3. 概要 (`about.html`)

- **役割**: 南陵祭のコンセプトや挨拶。
- **内容**:
  - 今年のテーマ解説。
  - 実行委員長挨拶。
  - パンフレット（PDF）へのリンク。

## 4. 開発者情報 (`about-us.html`)

- **役割**: クレジット表記。
- **内容**:
  - **制作**: 横浜南陵高校 コンピュータ科学部 (CSC)。
  - **メンバー**: 開発に関わった部員の一覧（ニックネーム可）。
  - **技術スタック**: 使用技術 (Firebase, Vanilla JS, Gemini etc.) の紹介。
  - **リンク**: Github や 部活の公式サイトへのリンク。

## 5. モバイルオーダー技術仕様 (`mobile-order-guide.html`)

- **役割**: システムの堅牢性を証明する「技術仕様書 (Whitepaper)」としての解説ページ。
- **ターゲット**: 校長、教頭、技術監査担当、保護者。
- **コンテンツ**:
  - **Architecture**: **GitHub Pages** (Frontend) + **Firebase** (Backend) のサーバーレス構成。
  - **Transaction Integrity**: **Cloud Functions** 経由でのみ注文を受け付ける仕様 (`allow create: if false`).
  - **Access Control**: **Deny by Default** のセキュリティルール。
  - **Lifecycle**: データ自動削除と、金銭リスクゼロ (No Payment Function) の保証。

## 6. UI/UX 共通方針

- **シンプルさ**: テキストと画像を読みやすく配置する「記事型」レイアウト。
- **ナビゲーション**: ページ上部に「戻るボタン」を配置し、ホームへの動線を確保。
- **ダークモード**: テキストの視認性を確保するため、白背景・黒文字（ライト）と黒背景・白文字（ダーク）のコントラスト比に配慮。
